# Pacific Fish — План разработки

## Стек технологий

| Компонент | Решение |
|-----------|---------|
| Фреймворк | **Next.js** (фронт + API routes) |
| ORM | **Prisma** |
| БД | **PostgreSQL** |
| Авторизация | **NextAuth.js (Auth.js)** |
| Хранение картинок | **Локально на сервере** (папка /uploads, Docker volume) |
| Telegram уведомления | **grammy** (встроено в Next.js API) |
| Контейнеризация | **Docker + Docker Compose** |
| Сервер | **Timeweb Cloud VPS** (~2GB RAM) |
| CI/CD | **GitHub Actions** |
| Reverse proxy + SSL | **Nginx + Let's Encrypt** |
| Рендеринг | **SSG + ISR** (статика + фоновая регенерация), App Router, Server Components |
| i18n | **next-intl** (серверный рендеринг переводов), языки: RU, EN, ZH |
| Домен | Купить + настроить DNS |

---

## План действий

### Этап 1 — Минимальный проект + Docker
1. Инициализировать Next.js проект (минимальная страница-заглушка)
2. Написать Dockerfile для Next.js
3. Написать docker-compose.yml (app + PostgreSQL)
4. Проверить что всё работает локально в Docker

### Этап 2 — Сервер и домен
5. Арендовать VPS на Timeweb Cloud
6. Настроить сервер (Docker, Docker Compose, SSH-ключи)
7. Установить и настроить Nginx (reverse proxy)
8. Купить домен, привязать DNS к серверу
9. Настроить SSL (Let's Encrypt через Certbot)

### Этап 3 — CI/CD
10. Настроить GitHub Actions: MR в main → сборка Docker-образа → деплой на сервер по SSH
11. Добавить секреты в GitHub (SSH-ключ, адрес сервера, переменные окружения)
12. Первый деплой, проверка всего пайплайна

### Этап 4 — Интернационализация (i18n)
13. Настроить next-intl (middleware, routing по /ru, /en, /zh)
14. Создать JSON-файлы переводов для каждого языка
15. Переключатель языков в layout

### Этап 5 — База данных и авторизация
16. Настроить Prisma + схему БД (News, Price, User, ContactRequest)
17. Настроить переменные окружения (.env)
18. Сделать авторизацию через NextAuth.js (Credentials — логин/пароль)

### Этап 6 — Бэкенд (API)
19. API routes: CRUD новостей (создание, редактирование, удаление, получение)
20. API routes: CRUD прайса
21. API route: загрузка картинок (сохранение в /uploads, путь в БД)
22. API route: форма обратной связи → отправка уведомления в Telegram через grammy

### Этап 7 — Админка
23. Страница логина
24. Страницы управления новостями (список, создание, редактирование)
25. Страницы управления прайсом
26. Загрузка картинок к новостям через форму

### Этап 8 — Публичная часть сайта
27. Страница новостей (список + детальная)
28. Страница прайса
29. Форма обратной связи (имя, телефон, сообщение → отправка на бэк → Telegram)
30. Общий layout, навигация
31. Настроить volume для /uploads (чтобы картинки не терялись при обновлении)
